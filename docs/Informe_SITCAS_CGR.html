1) Resumen Ejecutivo

Se implementó una plataforma institucional moderna para el SITCAS / CGR, con un frontend en Next.js y gestión de contenidos desde WordPress. El sistema permite publicar comunicados, documentos oficiales, galerías e información institucional; además de un formulario de afiliación con validaciones, carga de archivos y envío automático de correos.

La solución integra una arquitectura moderna, escalable y segura, con enfoque en la transparencia, la comunicación oportuna y la facilidad de administración de contenidos por parte del equipo sindical.

2) Objetivos del Proyecto

  • Modernizar la presencia institucional del sindicato con identidad visual coherente.
  • Centralizar la publicación de comunicados, documentos y galerías.
  • Facilitar la afiliación mediante formularios con validación y confirmación.
  • Asegurar la comunicación efectiva con afiliados y público.

4) Arquitectura y Tecnología

Componente Descripción
Frontend Next.js 16 (App Router), UI responsive y estilos institucionales.
CMS WordPress REST API con CPT y ACF para contenidos.
Correo SMTP (Nodemailer) con confirmación automática.
Archivos Vercel Blob para almacenamiento de formatos firmados.

5) Integraciones WordPress

  • comunicados: listado y detalle con PDFs.
  • galeria: fotos con ACF foto_galeria.
  • documentos_sitcas: documento, categoría, fecha.
  • directiva: cargo, correo, linkedin.
  • formato_afiliacion: archivo editable.
  • slider: contenido para portada.

6) Flujos de Trabajo y Operación

  • Publicación de contenidos: El equipo registra comunicados, galerías y documentos en WordPress.
  • Consumo en el frontend: La web consulta el REST API y renderiza la información en tiempo real.
  • Afiliación: El usuario descarga el formato, lo completa, lo sube y envía la solicitud.
  • Confirmación: Se envía correo al sindicato y correo de confirmación al solicitante.

7) Seguridad y Cumplimiento

Cumplimiento con la Ley 29733 de Protección de Datos Personales. La información recibida se usa exclusivamente para fines de afiliación y comunicación institucional.
  • Validaciones de formularios y confirmaciones de envío.
  • Variables sensibles gestionadas en entorno (no en código).
  • Control de tamaño de archivos adjuntos.

8) Rendimiento y Buenas Prácticas

  • Uso de miniaturas para galerías.
  • Carga diferida de imágenes (lazy loading).
  • Recomendación de subir archivos comprimidos.
